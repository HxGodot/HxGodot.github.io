HxGodot - A Haxe GDExtension for Godot 4

Haxe is an open source high-level strictly-typed programming language with a fast optimizing cross-compiler.

The Godot Engine is a free, all-in-one, cross-platform game engine that makes it easy for you to create 2D and 3D games.

"There always was interest from the Haxe community and I think it would be great it can work with Godot, but somehow it never materialized into anything."
- Juan Linietsky, Godot Creator and Technical Lead

He is right. Now, let's fix this, shall we?

HxGodot combines Haxe's hxcpp target with Godot 4's GDExtension mechanism to supercharge the way you build games with Godot. Nodes are written in Haxe, bundled as a GDExtension and behave like any other Node in your scenes(attach scripts, signals, etc).

At the same time you gain access to Haxe's wide range of libraries and tools inside of Godot. In general any library that is either pure Haxe or supports cpp's static linking should work. ➜ Checkout haxelib.

Under the hood we leverages Haxe's macro generation to create a full set of bindings so you can use interact with all classes/built-ins exposed in Godot 4. ➜ See the api-docs.

Features

  • Works with normal Godot4 builds. No extra runtime required.
  • No manual Memory Management. Garbage Collector included.
  • C++ Runtime Performance. Generally HxGodot should be more performant compared to using C#. Results may wary.
  • Benefit from the Haxe Ecosystem. ➜ Checkout haxelib
  • Parity with Godot-API where possible.

Supported Platforms

  • Windows: x86_64 (float_64)
  • Linux: x86_64 (float_64)
  • Macos: x86_64 [arm64] (float_64)
Apple Silicon [arm64] is technically supported but needs some manual tweaks atm.
